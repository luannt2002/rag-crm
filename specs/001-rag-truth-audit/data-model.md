# Phase 1 Data Model — RAG Truth Audit

Audit artifacts are versioned JSON/Markdown files (no new product DB tables). Schemas below
are the contracts between harness, grader, and reports. All files live under
`specs/001-rag-truth-audit/evidence/` unless noted.

## PipelineStage (truth-table row) — `evidence/truth_table.json`

```json
{
  "stage": "retrieve",                  // one of the 12 canonical stage names
  "grade": "L2",                        // "L1" | "L2" | "L3" | "disabled"
  "evidence": [
    {"kind": "trace", "path": "reports/rag_trace_60.json", "note": "candidates/top_k live"},
    {"kind": "code",  "path": "src/ragbot/orchestration/nodes/retrieve.py:42"}
  ],
  "missing_for_next_level": "no numeric target declared for recall@k",  // null if L3
  "label": "SU_THAT"                    // "SU_THAT" | "GIA_THUYET"
}
```

Validation: `grade` requires ≥1 evidence entry; `GIA_THUYET` rows cannot carry a grade.
12 stages fixed: ingest, chunk, embed, index, retrieve, rerank, grade, stats-format,
prompt-build, generate, guard, answer.

## RunRecord — `evidence/baseline_runs.json` (and per-step `evidence/step_<n>_runs.json`)

```json
{
  "run_id": "baseline-P-01-r03",
  "probe_id": "P-01",
  "iteration": 3,
  "corpus_version": {"chunks": 401, "max_updated_at": "...", "content_md5": "..."},
  "cache_status": "bypassed",           // MUST be "bypassed" or the batch is invalid
  "answer": "...",
  "numbers_extracted": [{"token": "26.000.000", "value": 26000000}],
  "verdicts": [
    {"value": 26000000, "class": "unsupported"}   // grounded | derived_valid | unsupported
  ],
  "answer_class": "sai_bia"             // chuẩn|chưa_chuẩn|thiếu|sai_bia|lệch|refuse_đúng|refuse_oan|deflect_oan
}
```

Aggregation per probe: `fabrication_rate = runs with class in {sai_bia, lệch} / N`;
fabricated-value distribution feeds the stray-number verdict (SỰ THẬT if ≥70% of fabricated
values string-contain a stray record number; refuted if ≤30%; else inconclusive).

## DecisionRecord — `evidence/decision_shell_entities.md`

Fields: options (a/b/c with cost S/S/L + risks from research.md D6), chosen_option, owner,
date, rejected_reasons. Remediation tasks MUST cite `chosen_option`.

## RemediationStep — `evidence/ladder.md` (append-only log)

Fields per step: step_no, change_id (single toggle/commit), red_test (path + failing-run
proof + passing-run proof), blast_radius (consuming flows + pin tests), pinned_rerun
(path to step_<n>_runs.json + delta table: chuẩn/sai_bia/lệch/thiếu/refuse_oan/deflect_oan
vs previous step), rollback_criteria (pre-declared), status (enabled | rolled_back).

Invariant: step N+1 cannot start until step N has `pinned_rerun` recorded.
