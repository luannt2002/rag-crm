
## ❹ Acceptance criteria (gate trước commit)

Coder team **PHẢI verify** load test JSON đạt 5 gate sau:

| Gate | Metric | Threshold | Source |
|------|--------|-----------|--------|
| **G1 sacred** | HALLU fabricate | **0** | Manual scan answer field cho 10 qaqc-tester turn (sacred trap) |
| **G2 quality** | Answer rate | ≥ 50% | `summary.answer_rate_pct` |
| **G3 refuse** | Refuse rate | ≤ 50% (qaqc trap → refuse OK) | `summary.refuse_rate_pct` |
| **G4 latency** | p95 ms | ≤ 25,000 ms | `summary.p95_ms` |
| **G5 cost** | Mean cost/turn | ≤ $0.0015 | `summary.mean_cost_per_turn` |

**Baseline reference** (production V16 `LOADTEST_3PERSONA_30Q_BASELINE_20260508_*.json`):
- Answer rate: 53-67%
- Refuse rate: 33-47%
- p95: 18,000-22,000 ms
- Mean cost: $0.00060-0.00065/turn
- HALLU: 0 sacred

## ❼ Failure handling

**Nếu G1 (HALLU) fail**:
- KHÔNG commit. Escalate `MAIN ADMIN HANDOFF: HALLU breach turn X.Y`
- Attach: full answer + retrieved chunks + sysprompt diff (nếu vừa edit)

**Nếu G2-G5 fail (quality/perf regression)**:
- LEAD CODER **rerun 1 lần** (network flake possible)
- Vẫn fail → escalate với 2 JSON file đính kèm (baseline production + failed local)

**Nếu Jina rerank fail (TimeoutError/401)**:
- Check `.env.coder` có `RERANKER_JINA_API_KEY` chưa fill
- Smoke test: `curl -H "Authorization: Bearer $RERANKER_JINA_API_KEY" https://api.jina.ai/v1/rerank -d '{...}'`
- Jina down → DEFER load test, document trong report
