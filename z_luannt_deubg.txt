1. Câu lệnh (Prompt) dành cho Claude Code

Bạn hãy copy chính xác đoạn sau đưa cho Claude Code thực thi:

    "Hãy giúp tôi gom (squash) toàn bộ các file migration trong thư mục /var/www/html/ragbot/alembic/versions/ lại thành một file migration duy nhất chuẩn bị cho release version. Hãy thực hiện tuần tự các bước sau:

    Bước 1 - Phân tích & Viết Note: Đọc toàn bộ các sub-file trong thư mục trên. Xác định thứ tự thực thi của chúng dựa vào revision và down_revision. Sau đó, tạo ra một file migration_flow_notes.md ghi chú lại toàn bộ luồng thay đổi của database schema một cách chi tiết nhất (bảng nào được tạo, cột nào được thêm/sửa/xóa qua từng file).

    Bước 2 - Tạo file tổng hợp: Tạo ra một file migration Alembic mới (ví dụ: [revision_id]_initial_release.py). Trong file này, hãy gộp tất cả các lệnh trong hàm upgrade() và downgrade() từ các file cũ thành một khối thống nhất. Đặt down_revision = None cho file này vì nó sẽ đóng vai trò là điểm bắt đầu (base).

    Bước 3 - Dọn dẹp: Sau khi tôi kiểm tra và xác nhận file tổng hợp mới đã đầy đủ data, hãy thực hiện lệnh xóa toàn bộ các file migration cũ đi, chỉ giữ lại file mới vừa tạo và file notes."

2. Cảnh báo quan trọng về mặt hệ thống (Database)

Claude Code có thể giúp bạn gom file code python rất tốt, nhưng nó không thể tự động sửa trạng thái database của bạn. Khi bạn thay đổi file trong thư mục alembic/versions, bạn phải đồng bộ lại database.

Sau khi Claude Code chạy xong và bạn chỉ còn lại 1 file duy nhất, bạn bắt buộc phải làm thêm bước này ở dưới database (PostgreSQL/MySQL/SQLite...) của bạn:

    Nếu đây là môi trường Dev/Local (Có thể xóa Data):

        Xóa hoặc Drop toàn bộ database hiện tại.

        Tạo lại database trắng.

        Chạy lệnh: alembic upgrade head (Nó sẽ chạy file duy nhất mà Claude vừa tạo).

    Nếu đây là môi trường đã có Data (Không thể Drop DB):

        Truy cập vào Database của bạn bằng DataGrip, DBeaver hoặc CLI.

        Tìm bảng có tên là alembic_version.

        Update giá trị version_num trong bảng đó thành cái ID (revision) của file mới mà Claude vừa tạo. Điều này giúp Alembic hiểu là database hiện tại đã "khớp" với file gộp mới này.

Cách "chuẩn" và an toàn nhất của Alembic (Không cần dùng AI gộp code)

Thực tế, để tạo một file release sạch trong Alembic, các kỹ sư thường dùng tính năng autogenerate thay vì ngồi gộp code thủ công. Nếu cách dùng Claude phía trên bị lỗi cú pháp, bạn có thể áp dụng luồng chuẩn này:

    Đảm bảo database hiện tại đã được update mới nhất: alembic upgrade head

    Xóa toàn bộ file trong /var/www/html/ragbot/alembic/versions/

    Xóa data trong bảng alembic_version ở database: TRUNCATE TABLE alembic_version; (hoặc DELETE FROM alembic_version;)

    Chạy lại lệnh tạo file mới từ đầu dựa trên Models hiện tại: alembic revision --autogenerate -m "Initial release version"

Cách này đảm bảo 100% file tạo ra sẽ khớp hoàn toàn với các Models SQLAlchemy hiện tại của bạn mà không sợ sai sót do AI gộp nhầm dòng code.

Quá tuyệt vời! Nếu đang ở môi trường Dev và có thể thoải mái reset, drop bảng rồi chạy seed lại thì mọi thứ đơn giản và triệt để hơn rất nhiều.

Dưới đây là prompt "một chạm" (all-in-one) cực kỳ chi tiết bằng tiếng Anh (để Claude Code hiểu logic code tốt nhất) mà bạn chỉ cần copy và paste thẳng vào terminal cho nó tự xử lý.
Copy đoạn Prompt này dán vào Claude Code:

    I am working on a Python RAG project in a dev environment. I need to squash all existing Alembic migration files in /var/www/html/ragbot/alembic/versions/ into a single base migration file for a release version. Since it's a dev DB, I will reset and re-seed it afterward.

    Please execute the following steps exactly in order:

    Step 1: Analyze and Document the Flow
    Read all python files in /var/www/html/ragbot/alembic/versions/. Determine the correct execution order by tracing the revision and down_revision variables. Once traced, create a markdown file named alembic_migration_notes.md in the alembic/ folder. This file must contain a highly detailed, chronological note of every schema change (which tables were created, which columns were added/modified/dropped, etc.).

    Step 2: Create the Squashed Migration File
    Create a brand new Alembic migration file in the versions folder (e.g., <new_revision_id>_initial_release.py).

        Consolidate all the upgrade() logic from the old files into this new file's upgrade() function, ensuring the chronological order is strictly maintained.

        Consolidate all the downgrade() logic into the new file's downgrade() function, maintaining the correct reverse-chronological order.

        Set down_revision = None in this new file.

    Step 3: Delete Old Files
    Once the new consolidated file and the notes markdown file are successfully created, completely delete all the old .py migration files in the versions folder. Only the newly created single migration file should remain.

    Step 4: Output the Reset Commands
    Finally, output the exact terminal commands I need to run to:

        Drop all tables in my database (or reset it).

        Run alembic upgrade head to apply this new squashed file.

        Run my database seed script.

Sau khi Claude Code chạy xong, bạn chỉ việc:

    Đọc qua file alembic_migration_notes.md xem có thiếu sót bảng nào không.

    Xóa sạch DB cũ (hoặc drop schema/tables).

    Chạy alembic upgrade head.

    Chạy script seed data của bạn.

Database sẽ sạch bong như mới với một file migration duy nhất chuẩn bị cho release!
