### F1 ¬∑ HIGH ¬∑ multi-bot ‚Äî Ingest idempotency key omits `record_bot_id`: second bot in a tenant ingesting the same URL is silently swallowed for 24h
- **FACT**: `for_ingest_document` hashes `("ingest", record_tenant_id, source_url, corpus_version)` ‚Äî no bot key (domain/value_objects/idempotency_key.py:40-52). `IngestDocumentUseCase.execute` passes `corpus_version=0` always (ingest_document.py:58-62). Dedup TTL = `DEFAULT_IDEMPOTENCY_TTL` = 86,400s (constants/_04_jwt_auth.py:94; idempotency.py:14-27). Contrast: `for_chat_message` DOES include `record_bot_id` (idempotency_key.py:23-37) ‚Äî the asymmetry is accidental, not designed.
- **Failure scenario**: tenant T, bot A ingests `https://example.com/pricing.pdf` at 09:00 ‚Üí key K registered ‚Üí job J_A. Bot B (same tenant, different corpus) ingests the SAME URL at 10:00 with the same document name: `existing` lookup is per-bot `(record_bot_id, tool_name)` ‚Üí `None` for bot B (ingest_document.py:73-76), `is_duplicate(K)` ‚Üí True ‚Üí returns **bot A's job_id** (ingest_document.py:76-85). Bot B's document row is never created; bot B's corpus silently misses the doc. The guard `existing is None and is_duplicate` (ingest_document.py:76) only rescues re-ingest of an already-existing row ‚Äî it cannot rescue a first-ingest for a second bot.
- **Fix direction**: add `record_bot_id` to `for_ingest_document` parts (1-line domain change + callers).

### F2 ¬∑ HIGH ¬∑ T1-smartness / dead-code ‚Äî The anti-hallucination final citation gate is built-but-never-wired
